## Valibot スキーマキャッシュ変換の計画

1. 目標仕様を整理し、`src/estree-transform.ts` に必要なユーティリティ関数（AST ガード、シリアライズ処理、ホイスティング補助など）の一覧を洗い出す。
2. `valibot-compiler:cache` という仮想モジュールを新設し、このモジュールにキャッシュ済みの `export const` を集約する。変換側は必要な識別子をここから import する形に書き換える。
3. `AstNode` の Union 判定だけでタイプガードできるよう配慮し、不要なユーティリティ（AST ガード、シリアライズ関数など）は導入しない。
4. 同一呼び出しをハッシュ（`hash-sum` 利用）で識別し、一貫した export 名を生成。元コードの重複箇所は仮想モジュールからの import を通じた参照に置き換える。
5. 実装後は型チェックやフィクスチャベースの変換テストで動作を検証し、最適化が確実に反映されたことを確認する。
